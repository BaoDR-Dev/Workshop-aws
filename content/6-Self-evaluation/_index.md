Throughout my internship from **April 17, 2026** to **July 30, 2026**, I had the opportunity to experience a real-world working environment, collaborating with team members on the **RecruitPro** recruitment system project. My core responsibilities included researching AWS cloud architecture solutions, building optimal infrastructure for the back-end application, configuring storage and networking services, as well as documenting the entire deployment process in the shared report system.

## 1. Learning Journey and Practical Experience

**Approaching new technologies:** Participating in the project helped me get familiar with and deepen my understanding of core AWS services including Amazon VPC (network design, public/private subnets), Amazon EC2 (virtual server management), Amazon RDS (relational database setup), Amazon S3 (secure file storage and access control), Elastic Load Balancer, Amazon CloudFront, and system monitoring via Amazon CloudWatch.

**Challenges and how I overcame them:** In the early stages, synchronizing connections between architecture layers — especially configuring network security between EC2 and the RDS database, as well as S3 bucket access control policies — caused confusion and errors. However, through persistent study of official technical documentation, careful analysis of system log error messages, and active communication with teammates, I gradually resolved these issues and developed a sharper troubleshooting mindset.

## 3. Weaknesses and Improvement Plan

Despite the encouraging results, I acknowledge there are still areas to improve going forward:

- **Time management:** At times, due to heavy workload or poor task allocation, some technical tasks were completed close to their deadlines. I need to apply more structured work methods such as Pomodoro or detailed priority lists.

- **Confidence in communication:** Presenting technical issues to a group or explaining solutions sometimes lacks flexibility and confidence. I will actively practice by speaking up more and participating in presentation activities.

- **Log reading speed and deep error analysis:** Pinpointing the root cause of errors in distributed systems sometimes takes too long. I plan to study monitoring tools more deeply and practice advanced debugging to sharpen my technical instincts.

## 4. Conclusion

The internship at the **RecruitPro** project concludes with many valuable hands-on lessons. While there is still much to learn ahead, the practical experience with AWS and the disciplined team spirit have given me a solid foundation and greater confidence to step into the IT job market in the future.
